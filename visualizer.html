Problem A: Dead Zones

When the TV (1920px wide) sits above two 1440p monitors (5120px combined), only the overlapping region allows cursor crossing. The edges outside the TV's width are dead zones — the cursor hits an invisible wall.

Mouse Warp fix: Detects cursor pressure at dead zone edges, then warps proportionally: ratio = (x - srcLeft) / srcWidth → newX = tgtLeft + ratio * tgtWidth

Problem B: Cursor Click Offset

On Wayland + NVIDIA, when the logical monitor origin is non-zero (TV at x=1591 instead of x=0), clicks can land several pixels away from the visible cursor tip.

Fix: Position TV at (0, 0) flush-left. Run dj video layout 3 to apply, then restart Wayland session.

Warp Formula

Proportional mapping ensures the full width of one row maps to the full width of another:

ratio = (cursorX - sourceRowLeft) / sourceRowWidth
newX = targetRowLeft + ratio * targetRowWidth

Example: cursor at 50% of TV → warps to 50% of bottom row (x=2560)
